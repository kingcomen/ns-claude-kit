# /prd — author a PRD as a GitHub issue

1. Ask only what's missing; do not over-ask.
2. Write a concise PRD: problem, scope, acceptance criteria, out-of-scope, affected scripts/records.
3. Create a GitHub issue with `gh issue create` and a feature branch `feat/<topic>-<issue#>`.
4. STOP. Wait for human confirm (Gate 1) before any code.
